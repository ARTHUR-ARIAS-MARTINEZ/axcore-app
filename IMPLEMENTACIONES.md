### 🧱 FASE 1: INFRAESTRUCTURA DE DATOS (CRÍTICO)
*   **Migración a la Nube**: Reemplazar `localStorage` por una base de datos real (Supabase o Firebase). Esto garantiza que el socio no pierda sus datos al cambiar de dispositivo.
*   **Blindaje de API Keys**: Implementar un sistema de backend (Proxy) para ocultar las llaves de Perplexity. Evita que usuarios técnicos roben el saldo de Arthur.
*   **Autenticación de Élite**: Sistema de login robusto con validación de correo y recuperación de contraseña.

### 💼 FASE 2: GESTIÓN DE NEGOCIOS Y COMPUMUNIDAD
*   **Multi-Control Clusters**: Configurar la arquitectura para manejar bloques de 5 a 10 gimnasios con llaves de API independientes.
*   **Tablero de Líderes (Leaderboard)**: Crear un ranking dentro de cada gimnasio basado en "Grasa Quemada Total" para fomentar la retención y competencia sana.
*   **Onboarding de Entrenadores**: Crear un manual rápido (1 página) para que los instructores del gym sepan cómo "recetar" AX-CORE a sus clientes.

### ⚖️ FASE 3: PROTECCIÓN LEGAL
*   **Términos y Condiciones**: Redacción de cláusulas de responsabilidad sobre consejos nutricionales y de salud.
*   **Aviso de Privacidad**: Protocolo de manejo de datos personales y biométricos bajo la ley mexicana.
